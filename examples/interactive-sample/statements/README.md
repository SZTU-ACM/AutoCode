# Interactive Sample

## 时间限制与空间限制

- 时间限制：2 秒
- 空间限制：256 MB

## 题目背景（可选）

本题展示 AutoCode 对交互题题面协议的推荐写法。

## 题目描述

交互器固定一个隐藏整数 `S`，满足 `1 <= S <= 100`。你需要通过若干次查询确定 `S`。

每次查询一个整数 `x` 后，交互器返回 `x` 与 `S` 的大小关系。确定答案后，输出最终答案并结束程序。

## 输入格式

本题为交互题，选手程序不会获得传统静态输入。

交互器内部隐藏参数满足：

- `1 <= S <= 100`
- `S` 在一次测试中固定不变
- 最多允许 `20` 次查询

## 输出格式

选手程序可以输出两类命令：

- 查询：`? x`，其中 `1 <= x <= 100`
- 最终答案：`! s`，其中 `1 <= s <= 100`

每次输出一条命令后必须立即刷新输出缓冲区。输出最终答案后，程序必须立即结束，不得继续输出或等待更多输入。

## 交互协议

交互开始时，judge 等待选手先输出一条命令。

若选手输出查询 `? x`：

- 当 `x < S` 时，judge 返回 `-1`
- 当 `x = S` 时，judge 返回 `0`
- 当 `x > S` 时，judge 返回 `1`

若选手输出最终答案 `! s`：

- 若 `s = S`，交互结束并判为正确
- 否则判为错误

以下行为会被判错或格式错误：

- 命令不是 `?` 或 `!`
- 查询或最终答案参数不在 `[1, 100]`
- 查询次数超过 `20`
- 输出最终答案后继续输出
- 提前结束输出、未及时刷新导致交互器无法读取、或读取 judge 响应后未按协议继续

## 样例

下面是一段可能的交互 transcript。以 `judge:` 开头的行表示交互器输出，以 `contestant:` 开头的行表示选手输出。

```text
contestant: ? 50
judge: -1
contestant: ? 75
judge: 1
contestant: ? 63
judge: 0
contestant: ! 63
```

## 说明

样例仅展示一种合法交互过程。实际测试中隐藏整数可能不同，选手必须根据每次 judge 响应动态决定下一步输出。
