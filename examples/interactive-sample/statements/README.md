# Interactive Sample

## 时间限制与空间限制

- 时间限制：2 秒
- 空间限制：256 MB

## 题目背景（可选）

本题为交互题示例，重点展示交互协议描述方式。

## 题目描述

交互器在开始时固定一个隐藏整数 `S`（`1 <= S <= 100`）。  
你需要通过提问尽快确定 `S` 的值。

你每次可以输出一个查询整数 `x`，交互器返回：

- `-1`：表示 `x < S`
- `0`：表示 `x = S`
- `1`：表示 `x > S`

当你确定答案后，输出 `! S` 结束交互。

## 输入格式

本题为交互题，不使用传统静态输入。  
交互器内部隐藏参数范围：

- `1 <= S <= 100`
- 最多允许查询次数：`20`

## 输出格式

交互过程输出格式：

- 查询：输出 `? x`
- 最终答案：输出 `! s`

每次输出后都必须立即刷新输出缓冲区。

## 样例

### 样例输入 #1

```text
-1
1
0
```

### 样例输出 #1

```text
? 40
? 70
? 55
! 55
```

## 说明

样例仅用于展示交互格式，不代表真实交互完整时序。实际评测以 interactor 协议为准，若输出格式错误、超出查询次数或未及时 flush，可能直接判错。
编写 brute 或模拟器时必须严格遵守同一交互协议，避免把“必须回应每次查询”的约束简化成离线一次性求解。
