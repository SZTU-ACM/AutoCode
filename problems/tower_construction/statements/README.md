# 积木高塔 (Tower Construction)

**时间限制**: 1.0 s  
**空间限制**: 256 MB

## 题目描述
小明有一排共 $N$ 块积木，每块积木都有一个宽度 $w_i$。
现在小明想把这些积木重新堆叠成若干座**新塔**。他必须遵守以下规则：
1. **顺序合并**：新塔必须由原数组中**连续**的一段积木合并而成。例如，原积木是 `[2, 1, 3]`，你可以把前两块合并成宽度为 $2+1=3$ 的塔，剩下的 `3` 独自成为一座塔。你不能改变积木的原始相对顺序，也不能跳过某些积木。
2. **严格递增**：从左到右，每一座新塔的宽度必须**严格大于**前一座塔的宽度。也就是说，如果建成的新塔宽度序列为 $B_1, B_2, \dots, B_k$，则必须满足 $B_1 < B_2 < \dots < B_k$。

你的任务是：计算在用完所有 $N$ 块积木的情况下，**最多**可以构建多少座塔？

*(注：合并并不是指物理上的堆叠，而是指将连续若干个数值相加作为一个新元素)*

## 输入格式
第一行包含一个整数 $T$ ($1 \le T \le 10^4$)，表示测试数据组数。
对于每组测试数据：
- 第一行包含一个整数 $N$ ($1 \le N \le 2 \cdot 10^5$)。
- 第二行包含 $N$ 个整数 $w_1, w_2, \dots, w_N$ ($1 \le w_i \le 10^9$)，表示积木的宽度。

保证所有测试数据的 $N$ 之和不超过 $2 \cdot 10^5$。

## 输出格式
对于每组测试数据，输出一个整数，表示能构建的最大塔数。

## 样例

**输入**
```text
3
5
1 2 3 4 5
5
5 4 3 2 1
6
2 1 2 10 3 1
```

**输出**
```text
5
2
3
```

## 样例解释
- **Case 1**: `1, 2, 3, 4, 5` 已经严格递增，直接作为 5 座塔。答案为 5。
- **Case 2**: `5, 4, 3, 2, 1` -> 方案：`5`、`10`（由 `4+3+2+1` 合并），共 **2** 座塔。
- **Case 3**: `2, 1, 2, 10, 3, 1` -> 方案：`2`、`3`（由 `1+2` 合并）、`14`（由 `10+3+1` 合并），共 **3** 座塔。
